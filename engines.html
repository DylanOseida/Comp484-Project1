Engines and Languages

Roblox Studio

Roblox Studio is the engine I’ve used the most. I have an almost complete game with 10 levels designed — only missing a proper UI, a lobby, and some technical polish. Roblox scripting uses Lua, which I’ve learned mainly through tutorials on YouTube (like The Dev King).

Roblox Studio is free to use, has a beginner-friendly UI, simple modeling tools, real-time collaboration with friends, and makes importing models easy.

Godot

I’ve explored Godot mostly through tutorials (Brackeys, for example). I only attempted one project and didn’t get far, but it was fun and feels like a strong next step after Roblox. Godot is open-source (like Blender), free, and has grown a lot in popularity recently.

It’s beginner-friendly but also has complex features similar to Unity.

Unity

I’ve followed a Udemy course and made some small mini-games in Unity. I’ve only worked in 2D, not 3D. It felt more challenging than Roblox or Godot, and while I never made a game fully on my own, I did start making my own assets for practice.

Unity is free (with paid tiers for professionals), uses C# as its main scripting language, and is a good step into intermediate game development.

Unreal Engine

I haven’t built anything myself in UE5, but I’ve seen many demos and videos of it in action. Unreal Engine is known for AAA-quality graphics, powerful 3D capabilities, and uses C++ and Blueprints for scripting.

It’s more complex than the engines I’ve used so far, but it’s an industry-standard engine used in high-end professional game development.
