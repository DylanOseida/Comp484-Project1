What is a Game Engine?

A game engine is a software framework designed to simplify the development of video games. Engines provide essential tools such as rendering, physics simulation, input handling, asset management, and scripting. Below are four engines I've explored, each with unique strengths and purposes. They range from beginner-friendly tools to industry-level platforms.

Engines Covered

  • Roblox Studio - beginner-friendly, uses Lua
  • Godot - open-source 2D/3D engine
  • Unity - popular for indie devs, uses C#
  • Unreal Engine - industry-standard, AAA graphics, uses C++ and Blueprints

Roblox Studio

Roblox Studio is the game engine I've used the most. I have a nearly complete game with 10 fun levels for players to enjoy. I am only missing some basic UI, a lobby, and some polish. Roblox scripting uses Lua, which I've mainly learned by watching YouTube tutorials (like The Dev King).

It's free, has a simple UI, supports real-time collaboration, and makes importing models easy. I would highly recommend new game developers to start off with this engine.

Godot

I found Godot through a YouTube channel called Brackeys. They have a great tutorial video on how to make a simple platformer game. I only attempted one project and didn't get far, but it felt like a strong next step after Roblox Studio. Godot is a FOSS, and starting to gain lots of popularity.

It's beginner-friendly but also has advanced features, making it a flexible engine for both 2D and 3D development.

Unity

I've followed a Udemy course and made some small mini-games in Unity. I've only worked in 2D, not 3D.

Unity is free (with pro tiers available), uses C#, and is widely used in indie game development.

Unreal Engine

I haven't started any projects myself in UE5, but I've seen many videos and it all looks very complex. Unreal is known for AAA-quality graphics and uses C++ and Blueprints.
